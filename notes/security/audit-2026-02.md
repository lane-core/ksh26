# Security Audit ‚Äî February 2026

Audit of ksh26 source (post-library-reduction, ~45K lines in src/cmd/ksh26/).
Focus: stack buffers, format strings, signal handlers, integer overflow.


## Findings

### Signal handler: malloc removed (fault.c) ‚Äî fixed

`sh_fault()` called `malloc(1)` to test heap availability before
cleanup on crash signals (SIGSEGV, SIGBUS, SIGILL). `malloc` is not
async-signal-safe (POSIX C2x 7.14.1.1) ‚Äî can deadlock if the signal
interrupts `malloc`/`free`.

**Fix**: Removed the `malloc(1)` probe. Always attempt `sh_done()`
for abort-class signals. In OOM scenarios, `sh_done()` may fail, but
deadlocking is strictly worse than an imperfect cleanup attempt.

File: `src/cmd/ksh26/sh/fault.c`, line ~123.


### Integer overflow guard: streval.c ‚Äî fixed

`stkalloc(sh.stk, ep->staksize * (sizeof(Sfdouble_t) + 1))` had no
overflow check on the multiplication. `staksize` is a `short`
(bounded by expression complexity, max ~32K), so practical overflow is
not possible, but defense-in-depth warrants the check.

**Fix**: Cast to `size_t`, guard with `SIZE_MAX` overflow check before
allocation.

File: `src/cmd/ksh26/sh/streval.c`, line ~166.


### Stack buffers ‚Äî clean

Audited all fixed-size stack buffers in the following files. All are
properly sized for their use or bounded by explicit length checks:

- `macro.c` ‚Äî `stakalloc`-based dynamic buffers, no fixed overflows
- `io.c` ‚Äî `char buff[IOBSIZE]` reads bounded by `sfrd` return value
- `lex.c` ‚Äî token buffers via `stkptr`, dynamically grown
- `edit/edit.c` ‚Äî `MAXLINE` bounds enforced on input
- `edit/vi.c`, `edit/emacs.c` ‚Äî edit buffers sized to `MAXLINE`


### Format strings ‚Äî clean

All `sfprintf`, `errormsg`, and `error` calls use literal format
strings. No user-controlled format strings found. Searched for:
- `sfprintf(*, %s)` patterns where the format itself is variable ‚Äî none
- `error(` with non-literal format ‚Äî none


### strcpy usage ‚Äî audited, clean

22 occurrences of `strcpy` across the ksh26 source. All either:
- Copy into buffers pre-sized with `strlen()` + allocation
- Copy short fixed strings (signal names, option names) into
  adequately sized buffers
- Use `stkalloc`/`stkseek` to ensure space before copy

No unbounded copies found.


### setuid/privilege handling ‚Äî adequate

`sh_init()` drops privileges correctly via `setuid(getuid())` when
privileged mode is off. The `SH_PRIVILEGED` option controls this.
No privilege escalation paths found.


## Out of scope

- sfio library internals (separate audit surface, will be abstracted)
- libast string functions (strmatch, strgrpmatch ‚Äî need separate audit)
- libcmd builtins (grep, chmod, etc. ‚Äî limited attack surface as builtins)
