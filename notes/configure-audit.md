# configure.sh Probe Hygiene Audit

Audit of all ~270 `#define`s generated by the replace-iffe-v2 monolith
(configure.sh, 8,806 lines). Each define checked for consumers in
`src/**/*.{c,h}` (excluding `src/cmd/INIT/`, `configure.sh`, and iffe
`features/` input files).

**Date**: 2026-03-18
**Branch**: new-build-sys
**Reference**: replace-iffe-v2 monolith (test oracle)

## Summary

| Classification | Count | Action |
|---------------|-------|--------|
| KEEP | ~175 | Retain probe, port to new system |
| SIMPLIFY | ~20 | Replace runtime probe with unconditional `#define` (C23 guarantee) |
| ELIMINATE | ~75 | Remove entirely (no consumers in source) |

**Bugs found**: 1 (`_mmap_worthy` never emitted — sfio mmap permanently disabled)

**Probe count reduction**: 56 probes → estimated ~45 after eliminating
dead-weight sub-probes from retained probes. The probe *functions* mostly
stay (they contain both kept and eliminated sub-probes), but the sub-probe
count within each function drops significantly.

## Platform scope decision

ksh26 targets:
- **Darwin** (macOS, arm64/x86_64) — primary development platform
- **Linux** (glibc, aarch64/x86_64) — primary deployment platform

Platforms NOT targeted (dead branches):
- SunOS / Solaris / illumos
- AIX (`_ALL_SOURCE` fallback ladder)
- HP-UX (`_lib_setpgrp2`)
- IRIX (`_lib_getuniverse`, `_lib_universe`)
- SCO / 386/ix (`_hdr_jioctl`)

The `probe_ast_standards` 10-branch cascade can be simplified to 2 branches
(Darwin + GNU/Linux). Branches 3-9 are dead. This saves ~280 lines.

Exception: `_lib_universe` and `_sys_universe` are consumed in `astconf.c`
but the code path only executes on systems where `universe(2)` exists
(IRIX). These defines will always be 0 on Darwin/Linux. Keep the probe
(cheap) but note it's vestigial.

## Bug: `_mmap_worthy` never emitted

`probe_ast_mmap` (lines 2853-3028) runs an mmap performance benchmark
and sets `_lib_mmap` on success. However, it never emits `_mmap_worthy`,
which is what `sfhdr.h` (line 68) and `sfseek.c` (line 275) actually
guard on. The original iffe `features/mmap` probe emitted `_mmap_worthy`
as a separate `tst` result; the native translation absorbed the benchmark
into `_lib_mmap` but forgot to also emit `_mmap_worthy`.

**Impact**: sfio mmap is permanently disabled even when the benchmark passes.
`#if !_mmap_worthy` is always true (0 is falsy → `!0` = true → `MAP_TYPE`
gets `#undef`'d).

**Fix**: When the mmap performance test passes, also emit
`#define _mmap_worthy 2`. When it fails, emit nothing (0 is the correct
default for "not worthy").

**Status**: Fixed on main (configure.sh line 2977). Build+test: 100/102
(path failures pre-existing).

These are pure iffe artifacts — the `nxt` mechanism finds native header
paths for AST's wrapper headers to `#include` without circular inclusion.
The native configure.sh doesn't use AST wrapper headers that need this
indirection. All `_nxt_*` defines have zero consumers in `src/`.

## Impact on probe count

The new system should implement only probes that produce KEEP or SIMPLIFY
defines. Entire probes that can be eliminated:

- `probe_ast_eaccess` — stub, guard only, no consumers
- `probe_ast_ndbm` — entire probe dead (no ndbm usage in ksh26)
- `probe_ast_random` — entire probe dead (arc4random used unconditionally)
- `probe_ast_sizeof` — 4 of 5 outputs dead; only `_ast_sizeof_long` survives (can fold into probe_ast_common)

All other probes survive but shed dead sub-probes (fewer `probe_hdr`,
`probe_lib`, `probe_mem` calls within each function).

**Estimated probe function count**: 56 → ~52 (4 entire probes eliminated)
**Estimated sub-probe count**: ~270 defines → ~195 (75 eliminated)
**Estimated compile invocations saved**: ~60-80 per configure run
