# ksh26 Architecture Notes

## ksh93 Internals
- Global state: `Shell_t sh` (defs.c) — single struct, AT&T legacy
- AST: `Shnode_t` union (shnodes.h), tagged by `tretyp & COMMSK`
- Eval: `sh_exec()` in xec.c — giant switch on node type
- Names: `Namval_t` + discipline stack (`Namfun_t` chain) in name.c/nvdisc.c
- Scoping: CDT viewpaths (`dtview()`) — sh.var_tree shadows sh.var_base
- Expansion: macro.c — tilde, param, cmdsub, arith, split, glob, quote removal
- Stack alloc: sh.stk (Stk_t) — bulk-freed at function/script boundaries

## Build System Architecture
- configure.sh → build/configure/driver.sh (probe primitives, batch helpers, orchestration)
- 58 probes in build/configure/probes/*.sh (manifest.sh registers them by tier)
- Tiers 0-7: standards → api/common/lib → system capabilities → fcntl → conf/limits → subsystems → ksh26/libcmd/pty
- Generated output: FEATURE headers, build.ninja, test-env.sh, run-test
- Nix wraps configure.sh in configurePhase, samu in buildPhase, tests in checkPhase

## Test Infrastructure
- run-test (generated by test-infra.sh): dual-mode (ninja capture / interactive live)
- Test contexts: default.sh (env), tty.sh (PTY wrapper), paths.sh (FHS hook), timing.sh (timeout hints)
- Signal reset: trap + $(kill -l) before exec (fixes nix-daemon SIGPIPE SIG_IGN)
- Process isolation: setsid helper compiled by configure.sh
- Result counting: pass+fail validated against stamp_count from build.ninja

## Key Probe Patterns
- _PROBE_STDERR: /dev/null for primitives, $LOGDIR/probe.log for delegates via probe_run()
- -include FEATURE/standards before -include stdio.h (glibc _GNU_SOURCE ordering)
- DEFPATH detection: detect_defpath() for NixOS command -p compatibility
- __FILE = FILE in ast_std.h for glibc 2.42 wchar.h compatibility
