## Context

The test harness runs 114 tests across 2 locales on both darwin and linux.
`run-test.sh` is a ~300-line POSIX shell script generated by configure.sh.
It sets up the test environment, runs the test under ksh, collects results,
and manages temp directories.

Current failure modes:
- Commands fail silently (`mkdir`, `date`, `rm` returning non-zero)
- Traps overwrite each other (cleanup trap clobbers signal trap)
- `set -e` is not used (too blunt — breaks legitimate non-zero exits)
- Errors in subshells (pipes, command substitutions) are invisible

## Goals / Non-Goals

**Goals:**
- Command failures in the test harness are immediately fatal with diagnostics
- Trap management supports nesting (test context files push their own traps)
- Self-contained POSIX shell — no external dependencies
- Clear ISC attribution for modernish-inspired concepts

**Non-Goals:**
- Replacing `set -e` (too coarse for test harnesses)
- Shell bug workarounds (modernish handles dozens; we target ksh+dash+bash)
- Changing test logic — only the harness, not the tests themselves
- Performance optimization (traps and hardening add negligible overhead)

## Decisions

### 1. `die` — fatal exit from any context

```sh
die() {
    printf 'FATAL: %s\n' "$*" >&2
    # Kill the top-level process to exit even from subshells/pipes
    kill -s PIPE "$$" 2>/dev/null
    exit 128
}
```

modernish uses a complex `_Msh_die` mechanism with IPC. Our version is
simpler: SIGPIPE to `$$` ensures the top-level shell exits even when `die`
is called from a subshell or pipe segment.

### 2. `harden` — make command failures fatal

```sh
harden() {
    for _h_cmd in "$@"; do
        eval "${_h_cmd}() { command ${_h_cmd} \"\$@\" || die \"${_h_cmd} failed (exit \$?): \$*\"; }"
    done
}
```

Usage: `harden mkdir cp rm date dirname` — wraps each command as a shell
function that calls `die` on failure. The function shadows the external
command but uses `command` to avoid recursion.

modernish's version is 472 lines with custom exit-code expressions,
tracing, environment manipulation. Ours is ~10 lines covering the common
case: any non-zero exit is fatal.

### 3. `pushtrap` / `poptrap` — stack-based traps

```sh
pushtrap() {
    # Push current trap action onto a stack variable, set new trap
    _sig="$1"; shift
    eval "_trap_stack_${_sig}=\"\$(trap -p ${_sig})
\${_trap_stack_${_sig}:-}\""
    trap "$*" "$_sig"
}

poptrap() {
    # Restore the most recently pushed trap
    _sig="$1"
    eval "_prev=\"\${_trap_stack_${_sig}%%
*}\"
_trap_stack_${_sig}=\"\${_trap_stack_${_sig}#*
}\""
    eval "$_prev"
}
```

modernish's version is 634 lines handling pseudo-signals, shell-specific
bugs, and subshell detection. Ours handles the common case: push a trap
before entering a test context, pop it when leaving.

### 4. Integration point

`safety.sh` is sourced early in `run-test.sh`, before any test context
files. Context files (`default.sh`, `tty.sh`, etc.) can use `pushtrap`
to add their own cleanup without clobbering the main cleanup trap.

```sh
# run-test.sh (early)
. "$TESTS/../lib/safety.sh"
harden mkdir cp rm date dirname cat

# tests/contexts/default.sh
pushtrap EXIT '_context_cleanup'
```

## Risks / Trade-offs

**[Hardened commands in test subshells]** → `die` uses `kill -s PIPE $$`
which signals the top-level shell. If a test intentionally runs a command
that fails (e.g., testing error paths), the hardened wrapper would `die`.
Mitigation: only harden commands used by the HARNESS, not by the TESTS.
The harness runs under `/bin/sh`; the tests run under the ksh being tested.

**[Portability of `trap -p`]** → `trap -p SIG` output format varies between
shells. POSIX specifies the format but some shells deviate. We only need it
to work with dash, bash, and ksh — all three produce `trap -- 'action' SIG`.
